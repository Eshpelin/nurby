# Overnight UX review — 2026-07-12

Method: 7 waves of "Pat" agents (a non-technical persona) drove the real product
through the browser against an isolated local stack (fresh DB `nurby_uxtest`,
API :8787, local ingestion + perception, mediamtx, a looping CCTV RTSP feed
with cars/cyclists/pedestrians, host Ollama with gemma3:4b). 54 scenarios
covering onboarding, cameras, rules, AI/VLM, people, vehicles, timeline,
search, shares, recordings, doctor, notifications, pairing, retention, users.
Every fix below was re-verified in the browser afterwards (12-point regression
pass, all green) plus 143 backend tests on the touched areas.

Environment gaps found during bring-up (not product code, but self-host-relevant):
`.venv-test` lacked `ultralytics` (object detection silently disabled: zero
detections, so no rule ever fired; the UI actively contradicted itself about it)
and `insightface` (face recognition disabled). Both installed. The silent
degradation is the product side of that lesson: F17/F65 fixes make the doctor
honest, but a "detection model unavailable" doctor check would close it fully.
